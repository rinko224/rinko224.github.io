# Chapter0 基础知识

## 1 数的表示方式
在计算机内部，所有数据都以二进制(0和1)的形式存储和处理。
### 1.1 无符号数(Unsigned Numbers)
最简单的表达方式，能表示所有的非负整数。
1、规则:所有二进制位都用于表示数值的大小。
2、范围:对于一个$n$位的无符号数，其表示的数据范围是$[0,2^n-1]$


### 1.2 有符号数(Signed Numbers)
为了表示数的正负，往往引入最高位作为符号位，其中0表示正数，1表示负数。
有符号数的表示方式主要有两种:
#### 1.2.1原码表示法(Sign-Magnitude Representation)
1、规则:最高位表示数的符号，剩下的位表示数值的绝对值。
2、特点:
1.优点:简单直观，便于理解。
2.缺点:
* 0存在两个表达方式:00000000和10000000。
* 算术运算较为复杂:在进行加减法时，需要单独处理符号位，硬件处理复杂。

!!! tip
    用原码来进行加减法，其对硬件造成的困扰主要在于:
    1、需要专门的比较器来确定哪个数的绝对值更大，以确定符号和谁减谁。
    2、没能统一加法和减法，需要根据符号位和比较器的结果确定使用加法器还是减法器。

#### 1.2.2反码表示法（Two's Complement Representation）
1、规则:正数的表达方式与原码完全相同，负数则将其对应的正数的所有位取反。
2、特点:
- 大致统一了加法和减法，但是仍比较复杂。
- 仍存在两个0:00000000和11111111。
  
!!! tip "利用反码进行加减法的流程"
    1.将所有的运算都视为两个数的加法，这里两个数均采用反码表示。例如:
    - 3 - 4 = 3 + (-4)
    - -2 - 3 = (-2) + (-3)
  
    2.两数反码直接相加（包括符号位），若最后在最高位产生了进位1，就将1加到最后的结果中。
    3.最终得到的结果就是结果的反码。
#### 1.2.3 补码表示法（Two's Complement Representation）
1、规则:正数的表达方式与原码完全相同，负数则将其对应的正数的所有位取反后加1。
2、特点:
- 统一了加法和减法，无需根据符号位和比较器的结果确定使用加法器还是减法器。
- 只存在唯一的0:00000000。
- 范围:对于一个$n$位的补码数，其表示的数据范围是$[-2^{n-1},2^{n-1}-1]$
